Module 4

The Human-in-the-Loop Gate

Pause before dangerous operations, require approval.

Why HITL?

Imagine a bouncer at a club. The AI wants to do something risky — rm -rf /tmp, write to auth.py, search the web. It stops and asks: "Should I let this through?"

🚪
The nightclub metaphor
Without HITL: AI can run any tool, anytime — like a club with no bouncer. With HITL: AI pauses at dangerous tools and asks permission — like a club with a strict bouncer checking IDs.
💻
execute
Shell commands — most dangerous. Always gated.
📝
write_file
Creates or overwrites files.
🔍
web_search
Uses API credits and shares your query.
🌐
fetch_url
Reads content from any URL.

Design the approval prompt

When a gated tool is called, the CLI shows an approval prompt with the tool name, arguments, and risk description.

Implement the interrupt handler

When a gated tool is called, LangGraph pauses and waits for a human decision.

1

Agent invokes a gated tool like execute or write_file.
2

Generate a human-readable description of what the tool will do.
3
Show approval menu
Display the tool call and [Approve] [Reject] buttons.
4

5

If approved, tool executes. If rejected, error is returned to agent.

Add --auto-approve flag

For CI and scripting, you can bypass all HITL gates with --auto-approve.

auto_approve logic
1if auto_approve: 2 # No interrupts - all tools run automatically 3 interrupt_on = {} 4else: 5 # Full HITL for destructive operations 6 interrupt_on = _add_interrupt_on()
Line 1-3 — When auto_approve=True, interrupt_on={} means no tools are gated. All tools run immediately.
Line 4-6 — When auto_approve=False (default), _add_interrupt_on() returns the full map of gated tools.
⚡
When to use --auto-approve
Use it in CI scripts, automated pipelines, or when you trust the AI completely. Never use it interactively with unfamiliar prompts.

Check your understanding

A user reports: "I ran with --auto-approve and the AI deleted my entire project!" What went wrong?
Exactly! --auto-approve bypasses every HITL gate. With it enabled, the AI can run rm -rf, write files, anything — without asking. That's the intended behavior for CI, but dangerous interactively.
The flag works exactly as designed — it bypasses all approvals.
Without HITL, the AI will run any tool the task requires, including dangerous ones.
With --auto-approve, no approval prompt is shown at all.

Conversations That Remember

What is state?

Stateless vs stateful: a walkie-talkie is stateless (each message is independent), a phone call is stateful (you remember what was said before).

📞 Stateless vs Stateful
You (stateless)
"fix the bug"
AI
"What bug? I don't know which bug you mean."
You (stateful)

🧠
Why state matters
Without state, every message is treated as independent. The AI can't remember you just described a bug. With state, the AI reads the full conversation history.
